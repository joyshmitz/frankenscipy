# fsci-linalg ↔ LAPACK / BLAS name mapping

Reality-check audit of `scipy.linalg.lapack` (480 Fortran routines,
147 distinct kernels) and `scipy.linalg.blas` (128 routines, 46
distinct kernels) against fsci-linalg's high-level public API.

**Decision (per fsci-linalg DISCREPANCIES.md):** fsci-linalg is built on
`nalgebra`, a pure-Rust dense-linear-algebra library. There is no
Fortran shim and Rust's monomorphization makes per-dtype routing
automatic. We therefore do NOT expose the LAPACK / BLAS namespace
directly. This document is the **migration table** for users porting
scipy code that calls into the low-level wrappers.

Generated by frankenscipy-uzht. Refresh with
`python3 crates/fsci-linalg/audit/audit_lapack.py`.

## Quick lookup

If you have scipy code calling `scipy.linalg.lapack.<X>routine(...)`
or `scipy.linalg.blas.<X>routine(...)` (where `<X>` is one of `s d c
z` for the dtype), strip the dtype prefix and find the kernel name in
the table below.

The kernel name is what scipy.linalg.lapack.get_lapack_funcs returns
internally; the migration target is the fsci-linalg high-level
function whose semantics match.

## Declined

- **Per-dtype suffixes** (`s`, `d`, `c`, `z`): Rust monomorphization
  removes the need. Same fn signature works for f32 / f64 / Complex32 /
  Complex64 (where fsci supports the dtype).
- **All "introspection" routines** (`get_lapack_funcs`,
  `get_blas_funcs`, `find_best_blas_type`): see DISCREPANCIES.md.
- **Workspace-query variants** (most LAPACK routines have a "query
  the optimal LWORK" calling convention): nalgebra manages workspace
  internally; no analogue.

## Long tail (123 LAPACK + 32 BLAS kernels not yet mapped)

Run the audit script to enumerate. The unmapped set is dominated by:

- **Hermitian / Hermitian-banded / Hermitian-tridiagonal eigenvalue**
  variants (`heev`, `hbev`, etc.) — fsci-linalg's `eigh` covers the
  standard case; banded / packed forms are fsci gaps.
- **Packed-storage triangular routines** (`pp*`, `tp*`, `pf*`,
  `tf*`) — fsci uses dense storage exclusively.
- **Reorderings** (`tgexc`, `tgsen`, `trsen`, `trsyl`) — niche.
- **Generalized SVD / generalized least squares** (`ggsvd`, `gglse`)
  — file as gap beads if requested.

## How to use this file

If your scipy code reads `from scipy.linalg.lapack import dgesv; r = dgesv(A, b)`,
the migration is `let r = fsci_linalg::solve(&A, &b, SolveOptions::default())?;`.
Tolerances and error semantics may differ — see the function-level
rustdoc for each fsci entry point.

If your code uses a kernel NOT in the tables above, file a gap bead
linked back to **frankenscipy-uzht** so this document can be expanded.
