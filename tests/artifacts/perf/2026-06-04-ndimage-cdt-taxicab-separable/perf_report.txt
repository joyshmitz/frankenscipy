perf: fsci-ndimage taxicab distance transform — brute-force O(F·B) -> separable chamfer O(N·ndim)
=================================================================================================

Target : crates/fsci-ndimage/src/lib.rs :: distance_transform_by_metric (Taxicab metric),
         shared by distance_transform_cdt(taxicab) AND distance_transform_bf(taxicab).
Lever  : City-block (L1) is an additive separable metric (Σ_axis |Δ_axis|). Replace the
         O(foreground · background) scan with a separable two-pass chamfer: per axis, a 1-D
         forward/backward min(d, neighbor + 1) sweep. New helper cityblock_distance_transform.

Scope / gating
--------------
Fast path only when metric == Taxicab && background nonempty. The no-background sentinel and
the chessboard metric (L∞ = max over axes; max-coupled, NOT additively separable) keep the
brute-force path. Euclidean is unaffected (its own Felzenszwalb fast path, commit c7b5ddc0).

Isomorphism proof (byte-identical f64)
--------------------------------------
- The two-pass 1-D sweep computes min_j (f[j] + |i-j|) exactly (forward = min over j<=i,
  backward = min over j>=i); composing axes yields min over background of Σ |Δ| exactly.
- All values are exact integers, so the result is byte-identical to the brute-force integer
  sum-and-min regardless of order.
- Empirically: 0 mismatches / 7248 cells across shapes {24},{10,10},{16,24},{5,7,9},{1,19},
  {4,4,4} × fg densities {20,40,60,80}%, comparing BOTH distance_transform_cdt and
  distance_transform_bf vs a verbatim L1 brute force via .to_bits().
