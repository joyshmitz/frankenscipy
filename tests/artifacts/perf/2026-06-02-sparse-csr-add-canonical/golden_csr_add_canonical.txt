isomorphism proof:
- Shape mismatch errors are unchanged because add_csr still calls ensure_same_shape before selecting the fast path.
- Noncanonical CSR inputs keep the old COO/BTreeMap path because the fast path requires sorted_indices and deduplicated metadata on both operands.
- Canonical inputs have at most one stored value for each row/column coordinate. The direct merge visits rows in increasing row order and columns in increasing column order, matching the canonical COO->BTreeMap->CSR output order.
- Floating-point operation order is preserved for canonical coordinates: each output value starts from 0.0, then adds lhs, then adds rhs_scale * rhs for overlapping entries. Single-sided rhs entries still multiply by rhs_scale before insertion. Single-sided lhs entries still perform 0.0 + lhs before zero filtering.
- Explicit zero results are dropped with the same value != 0.0 predicate as combine_coo.
- No sorting tie-breaker, random number generator, or tolerance behavior is introduced.
